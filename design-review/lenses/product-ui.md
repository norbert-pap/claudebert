# Product UI / App Lens

Apply when reviewing SaaS products, web apps, mobile apps, or any tool with user accounts and workflows.

## Evaluation Criteria

- **Orchestrated first use** leading to the outcome someone signed up for. Not a UI tour — a path to value
- **Empty states** guide toward the first action. An empty screen is a design failure unless it tells you what to do
- **Error states:** specific, helpful, human. But not jokey if it's frustrating — context and brand dependent
- **Does the UI feel fast?** Perceived performance matters as much as actual performance
- **Navigation learnable in one session?** If users need to "figure out" the nav, the nav failed
- **Goal-directed:** every screen serves a user goal (Cooper). If you can't name who this screen is for and what they want, the screen shouldn't exist

## Lens-Specific Provocations

- "A new user just signed up. Walk me through their first 60 seconds. Do they reach value, or do they stare at an empty dashboard?"
- "Show me an error state. Does it tell the user what happened AND what to do next?"
- "What's the most complex workflow in this app? Can someone learn it without documentation?"
- "Show me a modal. Does the user genuinely need context from the screen behind it, or should this be its own page? Is it blocking the UI when it doesn't need to?"
- "Replace every word in the UI with 'banana.' Can users still navigate by layout alone? If not, the design depends on copy, not structure."
- "Fill out the main form. How many taps to complete it? Is there a field that could be inferred, defaulted, or eliminated entirely?"
- "Show me a disabled button. Why can't I click it? Does the UI tell me what's missing, or do I have to guess?"
- "Open this on a phone. Are the touch targets big enough at the edges — not just the center? Can I tap every button confidently with one thumb?"
- "Are there any icon-only buttons without text labels? Would a new user know what each icon does without guessing?"
- "Who's the P95 user of this feature — the person using it 50x a day? Does the UI let them fly, or does it force them through the same guided path as someone using it once a week? Where's the progressive disclosure seam?"
- "A brand-new user just landed here. Do they reach value in 60 seconds, or did you design the happy path for someone who already understands the product?"
