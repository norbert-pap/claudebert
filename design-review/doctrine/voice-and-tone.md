# Voice & Tone — adapted from designpowers

Voice is who you are. Tone is how you say it right now. Voice stays constant. Tone adapts to context. Load during Layer 1 copy provocations or Layer 2 when evaluating interface copy consistency.

## Voice Attributes

Define 3-5 attributes for the project. For each, define "this, not that":

| Attribute | This | Not that |
|-----------|------|----------|
| Warm | Friendly, approachable | Sycophantic, emoji-heavy |
| Knowledgeable | Confident, precise | Academic, condescending |
| Direct | Clear, action-oriented | Blunt, cold, robotic |
| Encouraging | Celebrates effort | Patronising, fake-cheerful |

The "not that" column prevents drift. If DESIGN.md defines voice attributes, use those.

## Tone by Context

| Context | Feel | Example |
|---------|------|---------|
| Success | Celebration, warmth | "Done. Your changes are live." |
| Error | Calm, helpful, no blame | "We couldn't save that. Check your connection and try again." |
| Onboarding | Welcoming, confidence-building | "Let's get you set up. About 30 seconds." |
| Empty state | Inviting, action-oriented | "Nothing here yet. Create your first one." |
| Loading | Reassuring, brief | "Loading your data..." |
| Destructive | Clear, calm, reversible | "Delete this? You can undo for 30 days." |

## Vocabulary

| Always use | Never use |
|-----------|----------|
| Simple, specific verbs | Jargon, marketing-speak |
| "you" / "your" | "the user" |
| Active voice | Passive voice |
| Short sentences | Compound sentences with semicolons |

## Reading Level Targets

| Audience | Target |
|----------|--------|
| General consumer | Grade 6-8 |
| Professional tool | Grade 8-10 |
| Technical docs | Grade 10-12 |

## Review Checklist

- [ ] Every screen sounds like the same person wrote it
- [ ] Voice attributes present in the copy (check against DESIGN.md if defined)
- [ ] Tone matches the context (error copy is calm, success copy is warm)
- [ ] No jargon or marketing-speak in UI copy
- [ ] Reading level at or below target
- [ ] Copy works when read aloud by a screen reader
- [ ] Empty states have a clear CTA, not just "nothing here"
- [ ] Error messages explain what to do next, not just what went wrong

## AI Slop Copy Tells

- Em dash overuse (every other sentence)
- Generic hero copy ("Revolutionize your workflow")
- Everything in Sentence Case That Doesn't Need It
- Purple-prose sentences trying too hard to sound human
- Feature descriptions that could describe any product
